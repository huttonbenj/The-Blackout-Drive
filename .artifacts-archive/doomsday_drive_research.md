## ⚠️ Real Risks (Don't Ignore These)

### 1. The Gatekeeper Problem
macOS will quarantine the launcher binary by default. Windows SmartScreen will block unsigned executables. This is the #1 user complaint across all competitors. You need to either:
- Pay for code signing (~$100–$400/year for an EV cert)
- Or write bullet-proof instructions that walk users through Gatekeeper/SmartScreen bypass
- Or use a different launch mechanism (HTML + local server is often less blocked)

### 2. The "It's Just Free Software" Reddit Problem
Tech-savvy buyers will post "this is literally just Ollama and open source models on a thumb drive, you can do this for free." This is true. Your defense is:
- **Curation has value.** Freeze-dried food is just food you could dehydrate yourself. People still pay for it.
- Be upfront about the stack (builds trust, disarms critics)
- The survival content library is the real unique value — they can't easily replicate that themselves

### 3. ARM vs. x86 / OS fragmentation
The Ollama binary is architecture-specific. You need to ship Windows (x86_64), macOS Intel, and macOS Apple Silicon launchers. Test on all three. If it doesn't work on an M1 Mac, you'll get chargebacks.

### 4. Support volume
PortableMind has 4,000 units sold. Imagine even 1% of that needing support — that's 40 customers emailing you about a 2014 Windows laptop with 4GB RAM that can't run the model. **Set minimum hardware requirements clearly before purchase.** This is survivable if you're transparent upfront.
